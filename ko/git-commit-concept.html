Git 커밋(commit)이 뭔가요? 저장(Ctrl+S)과 뭐가 다를까

2026. 7. 12.

Git을 처음 배우면서 제일 헷갈렸던 단어 하나만 고르라면 단연 커밋(commit)이에요. "그냥 저장하는 거 아니야?"라고 생각했는데, 알고 보니 완전히 다른 개념이었어요.

저장과 커밋은 뭐가 다른가

저장(Ctrl+S)은 지금 상태로 파일을 덮어쓰는 거예요. 예전 상태는 사라져요.

커밋(commit)은 "이 시점의 상태를 사진처럼 찍어서 기록으로 남기는 것"이에요. 커밋을 여러 번 하면, 마치 앨범에 사진을 계속 쌓아두는 것처럼 언제든 예전 시점으로 돌아가거나 비교해볼 수 있어요.

비유하면: 저장은 원고를 계속 고쳐 쓰는 것이고, 커밋은 "이 버전은 완성됐으니 기록으로 남겨두자" 하고 사진을 찍어두는 것이에요.

커밋할 때마다 남는 정보

커밋을 하나 남길 때마다 아래 세 가지가 함께 기록돼요.

  1. 누가 변경했는지 (이름/이메일)
  2. 언제 변경했는지
  3. 무엇을 변경했는지 (커밋 메시지)

그래서 Git을 쓰기 전에 git config --global user.name과 user.email을 등록해야 해요 — 앞으로 만들 모든 커밋에 "이건 내가 한 거다"라는 서명이 남기 때문이에요.

흔한 오해: 이게 "저장소 초기화"인가요?

아니에요! git config는 "이 컴퓨터에서 커밋할 때는 항상 이 이름/이메일을 써라"는 전역 규칙일 뿐이에요. 저장소(repository)는 완전히 다른 개념으로, 특정 폴더 하나를 "여기서부터 이력을 관리하겠다"고 선언하는 거예요 (git init 또는 git clone으로 시작).

비유하면: config 설정은 "도장을 새로 파서 등록해둔 것"이고, 저장소를 만드는 건 "그 도장을 실제로 쓸 서류철(폴더)을 여는 것"이에요. 도장만 파두고 서류철은 하나도 안 연 상태로는, 아무 기록도 남길 수 없어요.
