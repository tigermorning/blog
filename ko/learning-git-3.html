GitHub 저장소 clone·commit·push 사용법 (3) — Git이 뭔지도 몰랐던 내가 하루 만에 블로그를 만들기까지

2026. 7. 12.

이번 3편에서는 실제로 저장소를 만들고, 내 컴퓨터로 가져오고(clone), 파일을 고쳐서 GitHub로 올려보내는(commit, push) 전체 흐름을 다룹니다.

← 이전 편 (2) GitHub 계정과 SSH 연결

1. GitHub에서 저장소 만들기

저장소(repository)는 파일들을 모아두는 폴더 + 변경 이력이라고 생각하면 돼요.

  1. GitHub 우측 상단 + 클릭 → New repository
  2. 저장소 이름 입력
  3. "Add a README file" 체크(선택)
  4. Create repository 클릭

2. 내 컴퓨터로 가져오기 (clone)

저장소는 GitHub 서버 위에만 있어서, 내 컴퓨터에서 직접 파일을 수정하려면 먼저 통째로 복사해와야 해요. 이걸 클론(clone)이라고 해요.

저장소 페이지의 초록색 Code 버튼 → SSH 탭에서 주소를 복사해두세요.

🎨 바이브 코딩으로: 복사한 주소를 Claude에게 붙여넣으며 "이 저장소를 내 문서 폴더에 클론해줘"라고 말하면 알아서 받아와줘요.

🔍 궁금하다면: 실제로 어떤 명령어일까?

git clone git@github.com:내아이디/저장소이름.git

3. 파일은 실제로 어디서, 어떻게 여나요?

  1. 파일 탐색기 열기: 윈도우 키 + E
  2. 주소 입력창에 저장소 경로 입력 후 Enter
  3. 수정하고 싶은 파일을 마우스 오른쪽 클릭 → 연결 프로그램 → 메모장
  4. 원하는 문장만 고치기. <p>, <h1>처럼 꺾쇠괄호로 된 부분은 건드리지 않기

⚠️ 파일을 더블클릭하면 안 돼요. .html 파일을 더블클릭하면 수정할 수 있는 화면이 아니라, 완성된 웹페이지가 브라우저에 열려버려요. 반드시 마우스 오른쪽 클릭 → 연결 프로그램 → 메모장으로 열어야 글자를 고칠 수 있어요.

수정을 마쳤으면 Ctrl + S로 저장하세요. 저장하지 않고 바로 닫기(X)를 누르면 안 돼요 — 메모장이 "변경 내용을 저장하시겠습니까?" 하고 물어보긴 하지만, 헷갈리지 않게 항상 닫기 전에 먼저 저장하는 습관을 들이는 게 좋아요.

4. 기록 남기고(commit) → 올려보내기(push)

여기까지는 아직 내 컴퓨터 안에서만 바뀐 상태예요. 실제 GitHub에 반영하려면 이 상태를 기록으로 남기고(commit), 온라인에 올려야(push) 해요.

🎨 바이브 코딩으로: 파일만 고쳐서 저장해두고, Claude에게 "방금 바꾼 내용을 커밋하고 push해줘"라고 말하면 나머지는 대신 처리해줍니다.

🔍 궁금하다면: 실제로 어떤 명령어일까?

git add 파일이름
git commit -m "무엇을 바꿨는지 메시지"
git push origin main

add는 "이 파일을 이번 기록에 포함시키겠다"고 표시하는 거고, commit은 그 상태를 스냅샷으로 남기는 거고, push는 그 기록을 GitHub 서버로 실제로 올려보내는 거예요.

커밋이 "저장"과 정확히 뭐가 다른지 더 궁금하다면 아래 글도 함께 보세요.

참고 글 Git 커밋이 뭔가요?
