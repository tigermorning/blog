2026. 7. 12.

이번 2편에서는 GitHub 계정을 만들고, 내 컴퓨터와 GitHub을 안전하게 연결하는 방법(SSH)을 다룹니다.

← 이전 편 (1) 개념과 기본 세팅

1. GitHub 계정 만들기

GitHub은 내 저장소를 인터넷에 올려두고 관리할 수 있는 사이트예요.

  1. github.com 접속
  2. 우측 상단 Sign up 클릭
  3. 이메일, 비밀번호, 사용자 이름(아이디) 입력
  4. 이메일로 온 인증 코드 입력해서 가입 완료

여기서 정한 사용자 이름(아이디)은 나중에 홈페이지 주소에도 그대로 쓰이니, 신중하게 정하는 게 좋아요. (저는 tigermorning으로 정했어요.)

2. SSH가 뭔데?

GitHub에 파일을 올리려면 "이 컴퓨터가 진짜 내 계정 주인이 맞다"는 걸 증명해야 해요. 매번 비밀번호를 치는 대신, SSH는 미리 만들어둔 열쇠 한 쌍으로 자동 인증을 해줘요.

  • 개인키(private key): 내 컴퓨터에만 보관, 절대 공유 X (도장 그 자체)
  • 공개키(public key): GitHub 계정에 등록해두는 것 (도장의 인주 샘플)

어떻게 "나"라는 걸 증명하나요?

비밀번호처럼 값 자체를 주고받는 게 아니에요. 접속할 때마다:

  1. GitHub이 랜덤한 문제(데이터)를 하나 던짐 — "이거에 서명해봐"
  2. 내 컴퓨터가 개인키로 그 자리에서 즉석 서명을 만들어서 보냄
  3. GitHub이 등록된 공개키로 그 서명이 맞는지 검증

서명은 매번 새로 만들어지는 일회용이라 도청당해도 재사용이 안 되고, 개인키 자체는 절대 컴퓨터 밖으로 안 나가요.

비유하자면: 서버가 무작위 문서 한 장을 주면서 "여기 서명해봐"라고 함 → 나는 나만 가진 도장(개인키)으로 찍음 → 서버는 등록된 인주 샘플(공개키)과 대조해서 확인. 도장 자체는 보여주지 않고, 찍힌 자국만 보여주는 셈이에요.

3. SSH 키 만들기

이미 키가 있는지 먼저 확인해볼 수도 있지만, 없다면 새로 만들면 돼요.

🎨 바이브 코딩으로: Claude에게 "내 컴퓨터에 SSH 키가 있는지 확인하고, 없으면 새로 만들어줘"라고 말하면 알아서 확인·생성해줘요.

🔍 궁금하다면: 실제로 어떤 명령어일까?

ssh-keygen -t ed25519 -C "내 이메일"

몇 가지 질문이 나오는데(저장 위치, 비밀번호 설정 등) 그냥 Enter만 계속 눌러도 기본값으로 만들어져요.

4. 공개키를 GitHub에 등록하기

  1. 만들어진 공개키(파일 이름 끝에 .pub가 붙은 것) 내용을 복사
  2. GitHub 우측 상단 프로필 → Settings
  3. 왼쪽 메뉴에서 SSH and GPG keys
  4. New SSH key 클릭
  5. Title에는 아무 이름(예: "내 컴퓨터"), Key 칸에는 복사한 공개키 통째로 붙여넣기
  6. Add SSH key 클릭

5. 연결이 잘 됐는지 테스트하기

🎨 바이브 코딩으로: Claude에게 "GitHub SSH 연결이 되는지 확인해줘"라고 말하면 테스트하고 결과를 알려줘요.

🔍 궁금하다면: 실제로 어떤 명령어일까?

ssh -T git@github.com

이런 메시지가 나오면 성공이에요.

Hi (내 아이디)! You've successfully authenticated, but GitHub does not provide shell access.

뒷부분의 "shell access"가 안 된다는 문구는 에러가 아니라 정상 응답이에요 — GitHub는 인증만 확인해주고, 실제 터미널 접속은 원래 안 시켜준다는 뜻이에요.

SSH가 어떻게 "나"라는 걸 증명하는지 원리가 더 궁금하다면, SSH는 어떻게 나를 증명할까? 글에서 공개키·개인키·서명 개념을 더 깊게 다뤘어요.

이제 내 컴퓨터가 GitHub과 안전하게 연결됐어요. 다음 3편에서는 실제로 저장소를 만들고, 파일을 주고받는(clone, commit, push) 방법을 다뤄볼게요.
