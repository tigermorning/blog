GitHub "잔디심기"가 뭔가요?

2026. 7. 12.

이 블로그를 만들면서 GitHub를 매일 들여다보다가, 개발자들이 "오늘도 잔디 심었다"는 말을 쓰는 걸 봤어요. 처음엔 진짜 식물 얘기인 줄 알았는데, 알고 보니 GitHub 프로필에 있는 그래프를 가리키는 말이었어요.

잔디심기란

GitHub 프로필 페이지에 가면, 지난 1년간의 활동을 작은 초록색 네모 칸들로 채운 달력 모양 그래프가 있어요. 정식 명칭은 Contribution graph(기여 그래프)인데, 활동한 날마다 칸이 초록색으로 채워지는 모습이 잔디를 한 칸씩 심는 것처럼 보인다고 해서 한국 개발자들 사이에서 "잔디심기"라는 별명이 붙었어요.

비유하면: 다이어리나 습관 트래커에 오늘 할 일을 했으면 스티커를 붙이는 것과 비슷해요. 매일 스티커(초록 칸)를 채워나가다 보면, 한눈에 "이 사람이 얼마나 꾸준히 활동했는지"가 달력 전체로 보이는 거예요.

어떤 활동이 칸을 채우나요

  • 커밋(commit): 코드나 파일 변경 사항을 저장소에 기록하는 것 — 가장 흔하게 잔디를 채우는 방법
  • Pull Request(PR) 생성
  • 이슈(issue) 등록
  • 코드 리뷰 참여

한 가지 주의할 점: 비공개(private) 저장소에서의 활동은 기본적으로 잔디에 안 보여요. 프로필 설정에서 "Include private contributions"를 켜야 초록 칸으로 (내용은 안 보이고 개수만) 반영돼요.

잔디만 열심히 채우는 게 의미가 있을까

개발자 커뮤니티에서는 "잔디 채우기 집착"에 대한 논쟁도 있어요. 매일 아무 의미 없는 빈 커밋(empty commit)만 찍어서 초록 칸을 억지로 채우는 경우도 있는데, 이건 실제 실력이나 성과와는 무관해요. 채용 담당자들도 대부분 잔디 개수 자체보다 어떤 저장소에, 어떤 내용으로 기여했는지를 훨씬 중요하게 봐요.

다만 저처럼 이제 막 시작한 입장에서는, 매일 조금씩이라도 저장소를 건드리는 게 바이브 코딩을 꾸준히 이어가는 동기부여가 되긴 하더라고요. 실제로 이 블로그 저장소에 커밋할 때마다 제 잔디도 하나씩 채워지고 있어요.
