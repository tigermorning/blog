GitHub Actions로 블로그 예약 발행 만들기 (5) — Git이 뭔지도 몰랐던 내가 하루 만에 블로그를 만들기까지

2026. 7. 12.

4편까지 만들고 나니, 새로운 고민이 생겼어요. 글을 여러 편 몰아서 써두고 한꺼번에 올리면, 나중에 애드센스 심사 같은 걸 받을 때 "이거 AI가 대량으로 찍어낸 거 아니야?"라고 보일 수 있다는 거였어요. 그래서 미리 써둔 글을 하루에 몇 개씩, 시간 간격을 두고 자동으로 발행해주는 시스템을 만들어봤어요.

← 이전 편 (4) GitHub Pages로 무료 홈페이지 만드는 법

1. 목표 정하기

  • 하루 최대 3개까지만 발행
  • 발행 간격은 6시간 (한국시간 09시 / 15시 / 21시)
  • 3개보다 많으면 자동으로 다음 날로 이월
  • 날짜를 일일이 계산할 필요 없이, 순서대로 큐에 넣기만 하면 되도록

2. 큐(대기열) 파일 만들기

발행 순서를 적어두는 파일을 하나 만들었어요. _drafts/queue.txt에 발행할 글의 경로를 한 줄씩 순서대로 적어두는 방식이에요.

🎨 바이브 코딩으로: 새 글을 다 쓰고 나면 Claude에게 "이 글 예약 발행 큐에 넣어줘"라고 말하면, 큐 파일에 순서대로 추가해줘요. 날짜를 계산할 필요가 전혀 없어요.

🔍 궁금하다면: 큐 파일이 실제로 어떤 모습일까?

ko/learning-git-5.html
ko/learning-git-6.html

이렇게 한 줄에 파일 하나씩, 발행 순서대로 적혀 있어요. 맨 위 줄이 다음에 발행될 글이에요.

3. GitHub Actions로 정해진 시간마다 실행하기

GitHub Actions는 "정해진 시간이 되면 자동으로 특정 작업을 실행해주는 기능"이에요. 제 컴퓨터나 Claude 세션이 꺼져 있어도, GitHub 서버가 알아서 실행해줘서 훨씬 안정적이에요.

🔍 궁금하다면: 실제로 어떻게 설정할까?

저장소 안에 .github/workflows/publish-drafts.yml 파일을 만들고, 언제 실행할지 cron이라는 시간 표기법으로 지정해요.

on:
  schedule:
    - cron: '0 0 * * *'    # 한국시간 09:00
    - cron: '0 6 * * *'    # 한국시간 15:00
    - cron: '0 12 * * *'   # 한국시간 21:00

GitHub 서버는 한국시간이 아니라 UTC(세계표준시) 기준이라서, 한국시간 09시를 표현하려면 9시간을 빼서 UTC 00시로 적어야 해요.

4. 실행될 때마다 할 일: 큐 맨 앞 글 하나만 꺼내서 발행

정해진 시간이 되면, 작은 스크립트가 실행돼서 이런 일을 해요.

  1. 큐 파일의 맨 위 한 줄만 읽기
  2. 그 글 파일을 _drafts에서 진짜 위치(ko/ 등)로 옮기기
  3. 큐 파일에서 그 줄 삭제 (나머지는 자연스럽게 다음 차례로)
  4. 커밋하고 push
  5. (덤으로) sitemap.xml에도 새 글 주소를 자동으로 추가

한 번 실행에 딱 하나만 처리하기 때문에, 아무리 큐에 글이 쌓여 있어도 하루 3번(3개)을 절대 넘지 않아요.

5. 저장소 권한 설정도 필요해요

GitHub Actions가 대신 커밋하고 push하려면, 저장소 설정에서 권한을 허용해줘야 해요.

  1. 저장소 Settings → 왼쪽 메뉴 Actions → General
  2. 맨 아래 Workflow permissions 항목에서 Read and write permissions 선택
  3. 저장

이 설정을 안 해두면, 자동화가 실행은 되는데 마지막에 push하는 단계에서 권한 오류가 나요.

6. 사람이 직접 발행한 글도 하루 개수에 포함해야 해요

여기서 한 번 실수할 뻔했어요. 이 자동화 시스템은 자기가 실행한 횟수만 셀 뿐, 제가 그날 직접 커밋해서 올린 글은 전혀 몰라요. 그래서 "오늘 이미 직접 3개를 올렸으니, 큐에 있는 글은 내일로 미뤄야 한다"는 판단은 사람(저와 Claude)이 따로 챙겨야 하는 부분이에요.

이 스크립트는 발행할 때마다 sitemap.xml도 자동으로 갱신하는데, sitemap이 왜 필요한지는 블로그 검색 노출(SEO) 기초 글에서 자세히 다뤘어요.

이제 글을 미리 여러 개 써두고 큐에 넣어두기만 하면, 나머지는 알아서 하루 3개씩 자동으로 나가요. 다음 6편에서는 실습해봤던 Git의 브랜치(branch)와 병합(merge) 개념을 다뤄볼게요.
