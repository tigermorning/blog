API가 뭔가요? AI랑 헷갈리지 않는 법

2026. 7. 12.

"API를 활용해서 뭘 한다"는 말을 듣고, 저는 순간 "그거 AI 얘기 아니야?"라고 생각했어요. 알고 보니 완전히 다른 개념이었어요.

API는 "웨이터" 같은 존재

API(Application Programming Interface)는 서로 다른 프로그램끼리 대화할 수 있게 해주는, 규칙이 정해진 창구예요.

레스토랑에 비유하면: 나(손님)는 주방(서비스)에 직접 들어가서 요리법을 몰라도 돼요. 웨이터(API)에게 정해진 방식으로 주문하면, 웨이터가 주방에 전달하고 결과를 다시 가져다줘요.

API ≠ AI

발음이 비슷해서 헷갈릴 수 있는데 완전히 다른 개념이에요.

  • AI(인공지능): 스스로 학습하고 판단하는 기술/기능 자체
  • API: 프로그램끼리 데이터를 주고받는 통신 방식 — AI든 아니든 상관없이 모든 서비스에 다 있음

API는 AI와 전혀 무관한 곳에도 훨씬 많이 쓰여요. 날씨 API, 지도 API, 결제 API 전부 AI와 상관없어요. "AI API를 활용한다"는 AI 기능을 갖다 쓰는 것이고, "API를 활용한다"는 아무 외부 서비스 기능이든 갖다 쓰는 훨씬 넓은 의미예요.

실제 예시: 국립국어원 오픈API

국립국어원은 표준국어대사전 오픈API라는 걸 제공해요. 프로그램이 "이 단어 맞춤법 맞아?"라고 정해진 형식으로 물어보면, 국립국어원 서버가 표준국어대사전을 검색해서 "맞음/틀림, 표준 표기는 이거임" 같은 답을 돌려줘요.

비유하자면: 도서관 사서 같은 느낌이에요. 서고(데이터베이스)에 직접 안 들어가도, 사서(API)에게 정해진 방식으로 물어보면 대신 찾아서 답을 갖다줘요. API마다 "이렇게 물어봐야 한다"는 고유한 질문 양식(API 문서)이 있는 것도, 사서마다 응대 방식이 다른 것과 비슷해요.

저는 번역 감수(맞춤법·띄어쓰기·외래어 표기 검토) 자동화에 관심이 있는데, 이런 API를 활용하면 사람이 매번 사전을 찾아보는 반복 작업을 프로그램이 대신할 수 있어요. 이 프로젝트 이야기는 나중에 별도 글로 자세히 다뤄볼게요.
