Git이 뭔지도 몰랐던 내가 하루 만에 블로그를 만들기까지 (4) — GitHub Pages로 진짜 홈페이지 만들기

2026. 7. 12.

이번 4편에서는 지금까지 배운 걸 가지고, 실제로 인터넷 주소로 접속되는 진짜 홈페이지를 만들어볼게요. 사실 지금 이 글을 보고 계신 tigermorning.github.io 사이트 자체가 이 방법으로 만들어진 거예요.

← 이전 편 (3) 저장소와 파일 주고받기

1. GitHub Pages가 뭔가요?

GitHub Pages는 GitHub이 무료로 제공하는 웹사이트 호스팅 기능이에요. 저장소 안에 index.html 파일 하나만 있으면, GitHub이 그 내용을 실제 웹사이트로 인터넷에 공개해줘요.

2. 저장소 이름을 특별하게 짓기

여기서 중요한 규칙이 하나 있어요. 저장소 이름을 정확히 내깃허브아이디.github.io로 만들면, 그 저장소는 루트 주소(예: tigermorning.github.io)로 바로 공개돼요.

그 외의 이름(예: hello-aiffel)으로 만들면, 주소 뒤에 저장소 이름이 붙어요 (예: tigermorning.github.io/hello-aiffel/).

참고: 저장소 이름은 나중에 바꿀 수도 있어요. 저는 처음에 tigermorning.github.io로 만들었다가 blog로 이름을 바꿨는데, 그 순간 주소도 tigermorning.github.io/blog/로 자연스럽게 바뀌었어요. 삭제하고 새로 만든 게 아니라, GitHub 저장소 Settings에서 이름만 바꾼 것이라 커밋 기록도 그대로 유지돼요.

3. index.html 파일 만들기

저장소 맨 위(루트)에 index.html이라는 이름의 파일이 있어야, GitHub이 그걸 홈페이지 첫 화면으로 인식해요.

🎨 바이브 코딩으로: Claude에게 "간단한 인사말이 담긴 index.html 파일을 만들어줘"라고 말하면 알아서 만들어줘요.

🔍 궁금하다면: 실제로 어떤 내용일까?

아주 기본적인 형태는 이런 모습이에요.

<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <title>내 이름</title>
</head>
<body>
  <h1>안녕하세요!</h1>
</body>
</html>

<h1> 태그 사이의 글자가 화면에 큰 제목으로 보이는 부분이에요. 이 글자만 원하는 내용으로 바꿔도 이미 나만의 홈페이지가 완성돼요.

4. 커밋 → push → 몇 분 기다리기

3편에서 배운 것과 똑같아요. 파일을 만들었으면 기록을 남기고(commit), GitHub로 올려보내면(push) 돼요.

🎨 바이브 코딩으로: Claude에게 "이 파일 커밋하고 push해줘"라고 말하면 끝이에요.

push가 끝나면 바로 사이트가 뜨는 건 아니고, GitHub이 페이지를 만드는 데 1~2분 정도 걸려요. 그 사이에 접속하면 404 에러가 뜰 수 있는데, 조금 기다렸다가 다시 새로고침(강력 새로고침: Ctrl+Shift+R)해보면 돼요.

여기까지가 Git·GitHub·GitHub Pages의 기본 흐름 전체예요. 준비물조차 없던 컴퓨터에서 시작해서, 이제는 실제로 인터넷에 공개된 내 홈페이지를 갖게 됐어요. 다음 5편에서는 조금 더 나아가서, 글을 미리 써두면 자동으로 하루 몇 개씩 예약 발행되는 시스템(GitHub Actions)을 만들어볼게요.
