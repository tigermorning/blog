오늘 수업은 "바이브 코딩으로 웹 만들기"였고, 1·2교시가 작업대를 차리고 확인하는 시간이었다면, 3교시는 그 작업대 위에 처음으로 산출물을 올려보는 시간이었어요. 이번 교시에서 처음 만든 건 my-first-web 폴더와 index.html 파일이었는데, 이 뒤로도 실습이 계속 이어져요. 그중 이 간단한 첫 단계에서도 실수를 하나 했고, 그 실수 덕분에 오히려 제대로 이해하게 된 게 있어서 그 얘기부터 해볼게요.

물결표(~) 하나 때문에 폴더가 엉뚱한 곳에 생겼어요

수업 자료에는 "프로젝트를 모아둘 위치에서 실행하세요. 예를 들어 문서 폴더 아래에 만들고 싶다면 먼저 cd ~/Documents로 이동합니다"라고 적혀 있었어요. 저는 이 ~가 뭔지 모른 채로 그냥 건너뛰고 바로 mkdir my-first-web을 쳤어요.

결과는 폴더가 제 계정의 가장 바깥(C:\Users\user) 바로 밑에 만들어진 거였어요. 문서 폴더 안에 만들고 싶었는데, 그게 아니었죠. 알고 보니 ~는 "내 홈 폴더"를 가리키는 줄임 기호였어요. 그러니까 cd ~/Documents는 "내 홈 폴더 안의 Documents 폴더로 이동해라"라는 뜻이고, 이 단계를 건너뛰면 지금 서 있는 곳(이 경우 홈 폴더 바로 밑)에 그냥 만들어지는 거였어요.

결국 잘못 만든 폴더는 지우고, 이렇게 다시 했어요.

cd C:\Users\user\Documents
mkdir my-first-web
cd my-first-web
pwd

pwd는 "지금 내가 어느 폴더에 서 있는지"를 알려주는 명령어예요. 컴퓨터의 모든 프로그램은 "지금 나는 어디 서 있다"는 책갈피를 하나씩 들고 있는데, cd는 그 책갈피를 옮기는 것이고 pwd는 "지금 책갈피가 어디 꽂혀 있어?"라고 물어보는 것이었어요. 결과가 C:\Users\user\Documents\my-first-web로 나와야 제대로 된 위치라는 뜻이에요.

touch가 안 먹혀요 — 또 다른 통역사 문제

수업 자료의 다음 명령은 touch index.html이었어요. 저는 PowerShell을 쓰고 있어서 이 명령어가 그대로 안 통했어요.

touch는 원래 유닉스·리눅스·맥 계열 명령어예요(맥이 이걸 갖고 있는 건 macOS 자체가 유닉스 기반이라서예요). 원래 뜻은 "이 파일을 건드려서 마지막 수정 시간만 업데이트해라"인데, 파일이 아예 없는 상태에서 쓰면 일단 파일을 새로 만들고 나서 시간을 찍어줘요. 그래서 실무에서는 "빈 파일 빨리 만들기" 용도로 많이 써요.

그런데 PowerShell은 유닉스 계열이 아니라서 이 단어 자체를 몰라요. 수업 자료에도 다행히 대안이 적혀 있었어요 — Windows PowerShell에서는 New-Item index.html을 쓰면 돼요. 정확히 같은 결과(빈 index.html 파일 하나)가 만들어져요.

AI에게 첫 파일 생성을 맡기기

이제 my-first-web 폴더 안에서 claude를 실행하고, 이렇게 부탁했어요.

index.html 파일에 브라우저에서 바로 열 수 있는 첫 화면을 만들어줘.
조건:
- 한국어 문서로 설정해줘.
- 큰 제목에는 [내 이름 또는 역할]을 넣어줘.
- 본문에는 [내 도메인]에서 AI를 활용해 보고 싶은 일을 한 문장으로 넣어줘.

여기서 중요한 문장이 하나 있었어요. "이 요청은 코드를 대신 써 달라는 요청이지만, 책임까지 넘기는 요청은 아니다"라는 거예요. 이게 정확히 1편에서 이해한 "루프"예요 — 자연어로 지시했다고 끝나는 게 아니라, AI가 만든 결과를 제가 직접 읽고 판단하는 단계가 반드시 따라와요.

AI가 만들어준 결과, 그리고 검토

제 도메인(영상 자막·SDH 감수)에 맞춰서 이런 결과를 받았어요.

<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>나의 첫 웹페이지</title>
</head>
<body>
  <h1>AI 에이전트를 공부하는 Tigermorning</h1>
  <p>영상 자막(한국어) 감수를 AI로 자동화하는 일에 AI를 활용해 보고 싶습니다.</p>
</body>
</html>

이 코드 자체가 검토 대상이라는 게 중요해요. index.html은 결국 텍스트 파일이라서, 저도 이 글자들을 그대로 읽고 판단할 수 있어요 — 특별한 도구 없이도요. 검토 질문 세 개에 직접 답해봤어요.

  • 제목이 내 역할이나 도메인을 드러내는가: 네, "AI 에이전트를 공부하는 Tigermorning"이라고 되어 있어요.
  • 본문 문장이 너무 넓거나 추상적이지 않은가: "영상 자막(한국어) 감수"라고 구체적인 도메인이 들어가 있어서 괜찮았어요. 만약 그냥 "AI를 잘 쓰고 싶습니다" 정도였다면 너무 넓었을 거예요.
  • 브라우저에서 열었을 때 글자가 보이는가: 네, 확인했어요.

산출물 확인하기 — 더블클릭 vs 메모장

index.html을 확인하는 방법은 두 가지예요. 더블클릭하면 브라우저가 이 텍스트를 해석해서 그림으로 그려주고, 오른쪽 클릭 → 메모장으로 열면 텍스트 원본 그대로 보여줘요. 저는 더블클릭해서 실제 화면으로 확인했어요.

  • my-first-web 폴더가 있다 — 확인
  • index.html 파일이 있다 — 확인
  • 브라우저에서 제목과 본문이 보인다 — 확인
  • 내 도메인 문장으로 바꿨다 — 확인

폴더 위치를 착각했던 것부터 시작해서, 결국 진짜 파일 하나를 만들고 브라우저로 확인하는 것까지 왔어요. 작은 파일 하나지만, 1·2편에서 이해한 것들(작업대, 도구, 텍스트 파일, 루프)이 실제로 처음 쓰인 순간이었어요. 3교시 실습은 여기서 끝이 아니라 계속 이어지는데, 그 다음 내용은 이어서 정리할게요.
