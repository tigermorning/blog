캐시(Cache)가 뭔가요? 사이트가 안 바뀌어 보일 때

2026. 7. 12.

분명 사이트를 고쳐서 올렸는데, 새로고침을 해도 예전 화면이 계속 보이는 경험 있으신가요? 저도 블로그를 만들면서 여러 번 겪었어요. 범인은 대부분 캐시예요.

캐시란

캐시는 브라우저가 "이 사이트 아까 봤으니까, 다시 인터넷에서 안 받아오고 저장해둔 걸 그냥 보여줄게"라고 알아서 저장해두는 임시 복사본이에요.

비유하면: 매번 프린터까지 걸어가서 새로 뽑는 대신, 책상 위에 어제 뽑아둔 복사본을 그냥 다시 보는 것과 비슷해요. 문제는 원본(진짜 최신 사이트)이 바뀌었는데도, 책상 위 복사본(캐시)이 예전 것이라서 바뀐 걸 못 보고 예전 화면만 계속 보이는 경우가 생겨요.

해결법 (쉬운 순서대로)

  1. 강력 새로고침: 그 페이지에서 Ctrl + Shift + R — 진짜로 새로 받아오라고 강제하는 것
  2. 시크릿 창으로 열기: 크롬 기준 Ctrl + Shift + N → 새로 뜨는 창에 주소 다시 입력. 시크릿 창은 아예 캐시를 안 쓰는 창이라 확실해요
  3. 캐시 완전히 삭제: Ctrl + Shift + Delete → 팝업에서 "캐시된 이미지 및 파일" 체크 → 삭제
  4. 그래도 안 되면 다른 기기(핸드폰 등)로 확인 — 서버 문제인지 내 컴퓨터 문제인지 구분하는 가장 확실한 방법

참고로 서버 쪽이 실제로 최신 상태인지는, 개발자라면 curl 명령어나 API로 직접 확인할 수도 있어요. 저는 GitHub Pages를 만들면서 이 문제를 자주 겪었는데, 그 과정은 시리즈 4편에도 잠깐 나와요.
